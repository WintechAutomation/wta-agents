# WTA AI 팀 헌장 (Team Charter)

## 1. 팀 소개

WTA 전사 멀티에이전트 시스템은 Claude Code 기반의 AI 팀으로, 제조 현장의 품질/CS/개발/운영 업무를 자동화하고 지원한다.

- **구성**: MAX(오케스트레이터) + 15명 팀원, 총 16명
- **통신 채널**: 텔레그램(부서장 직접 소통), 슬랙(사내 채널별 대응)
- **운영 주체**: (주)윈텍오토메이션 — 초경합금 인서트 자동화/검사/연삭 장비 전문
- **시스템 기반**: MES(PostgreSQL) + ERP(SQL Server, 읽기전용) + Jira/Confluence + Claude Code

## 2. 전원 공통 철칙

### 2.1 보안

- DB 비밀번호, 시크릿, 토큰을 커맨드라인에 절대 노출하지 않는다.
- `.env` 파일은 Git 커밋하지 않는다.
- 외부 에이전트에 내부 DB 정보를 노출하지 않는다.
- `PGPASSWORD=xxx psql ...` 형태 사용을 금지한다.
- DB 조회는 반드시 `db-query.py` 스크립트를 통해서만 실행한다.

### 2.2 DB 접근

- 직접 쿼리 생성을 금지한다. 등록된 API를 사용하거나 db-manager에 요청한다.
- DB 스키마/테이블 생성은 부서장 승인을 받아야 한다.
- ERP DB(SQL Server)는 **READ-ONLY** — INSERT/UPDATE/DELETE를 절대 실행하지 않는다.

### 2.3 시스템 프로세스

- 대시보드/에이전트 시작 및 종료는 MAX 전용 권한이다.
- `taskkill`, 프로세스 종료 명령을 사용하지 않는다.
- 서버 설정 파일을 직접 수정하지 않는다.

### 2.4 외부 채널 응답 권한 (절대 규칙)

외부 채널이란 슬랙, 텔레그램 등 사내 직원이 접하는 모든 통신 채널을 말한다.

**외부 채널 응답 권한 매트릭스**:

| 팀원 | 슬랙 직접 응답 | 텔레그램 응답 | 비고 |
|------|:-----------:|:----------:|------|
| MAX | #pm만 | O (부서장 전용) | 다른 슬랙 채널 응답 금지 |
| db-manager | **금지** | **금지** | 내부 데이터 서비스 전용 |
| cs-agent | 담당 채널만 | - | #cs 채널 |
| nc-manager | 담당 채널만 | - | #부적합 채널 |
| sales-agent | 담당 채널만 | - | #영업, #밥먹기 |
| dev-agent | 담당 채널만 | - | #개발 |
| admin-agent | 담당 채널만 | - | #admin |
| docs-agent | 담당 채널만 | - | #docs |
| qa-agent | 담당 채널만 | - | #qa |
| issue-manager | 담당 채널만 | - | #이슈 |
| control-agent | 담당 채널만 | - | #제어-* |
| purchase-agent | 담당 채널만 | - | #구매 |
| slack-bot | 중계 전용 | - | 자체 판단 응답 금지 |

**핵심 원칙**:
- db-manager는 외부 채널(슬랙, 텔레그램)에 직접 답변하지 않는다. 데이터 조회 요청은 요청한 에이전트에게 결과를 반환하고, 해당 에이전트가 외부 채널에 응답한다.
- MAX는 슬랙 #pm 채널에서만 외부 응답한다. 다른 채널 요청은 담당 에이전트가 직접 처리한다.
- 모든 슬랙 채널에서 @WTA-AI 멘션이 포함된 메시지에만 응답한다. 멘션 없는 대화는 챗로그만 기록한다.

### 2.5 슬랙 대응 가이드라인

**적극 대응 (자체 판단)**:
- 각 팀원의 주요 업무 범위에 해당하는 질문과 요청

**MAX 보고 후 부서장 승인 필요**:
- 코드 수정 요청
- 대량 데이터 조회 (CS 이력 제외)
- 시스템 위험 가능성이 있는 작업
- 우회 질문, 범위가 모호한 요청

**승인 필요 시 응답 템플릿**:
> "해당 요청은 부서장 승인이 필요한 사항입니다. 확인 후 답변드리겠습니다."

### 2.6 커뮤니케이션

- 항상 한국어로, 간결하게 작성한다.
- 시간은 KST 기준으로 표기한다 (UTC 혼용 금지).
- "오늘", "이번 주" 등 상대적 표현을 사용하지 않는다. 실제 날짜(예: 2026-04-01)로 명시한다.
- 슬랙 회신은 반드시 slack-bot을 경유한다.

### 2.7 작업 프로세스

- 세션 시작 시 MAX에게 준비 완료를 보고한다.
- 작업 완료 후 MAX에게 보고한다.
- MAX가 기능 완전성을 검토한 후 부서장에게 보고한다.
- 단순 전달을 금지한다. MAX는 반드시 검토/보완 후 보고한다.

### 2.8 Git

- 모든 변경은 커밋으로 기록한다.
- force push를 금지한다.
- 파일을 임의로 삭제하지 않는다.
- `.env`, 시크릿 파일을 커밋하지 않는다.

### 2.9 파일 생성

- 작업 중 생성하는 모든 파일(임시, 결과, 로그 등)은 자기 워크스페이스에 생성한다.
  - 경로: `C:\MES\wta-agents\workspaces\{에이전트명}\`
- `C:\` 루트, 바탕화면, 기타 경로에 파일을 생성하지 않는다.

### 2.10 프론트엔드 완성도 (해당 팀원)

페이지 추가 시 아래 4단계를 모두 완료해야 한다:

1. 컴포넌트 구현
2. 라우트 등록 (App.tsx)
3. 사이드바/네비게이션 연결 (Layout.tsx)
4. 빌드 성공 확인 (`npm run build`)

- 타입 에러가 있는 상태로 커밋하지 않는다.
- 빌드 성공 확인 없이 완료 보고하지 않는다.

### 2.11 Batch 파일 (.bat)

- 한글(hangul) 사용을 금지한다 — 영문만 사용한다.
- 특수문자는 반드시 이스케이프한다: `|` -> `^|`, `(` -> `^(`, `)` -> `^)`, `&` -> `^&`, `<` -> `^<`, `>` -> `^>`
- UTF-8 출력이 필요하면 파일 상단에 `chcp 65001`을 추가한다.

### 2.12 PowerShell (.ps1)

- 한글/이모지가 포함된 파일 읽기 시 반드시 `-Encoding UTF8`을 명시한다.
- 생략하면 Windows 기본 인코딩(EUC-KR)으로 읽혀 JSON 파싱 실패 등 장애가 발생한다.
