# ERP 현재고현황 및 구매진행현황 — 장비유형 분류 기준

> 담당: 김근형 | 최종 수정: 2026-04-10 | 적용 대상: `reports/김근형/erp_data.json`

## 목적
ERP 현재고현황 및 구매진행현황 리스트에서 품목별 장비유형 분류가 누락·오류 없이 유지되도록 하는 표준 규칙.
데이터 재생성 시 이 규칙을 반드시 적용하여 이전 수정 내역이 리셋되지 않도록 한다.

---

## 1. 장비유형 분류 규칙

### 1-1. 프로젝트명 기반 장비유형 매핑
발주이력의 프로젝트명에서 장비유형을 추출할 때 다음 키워드를 사용한다.

| 장비유형 | 매칭 키워드 |
|----------|------------|
| 프레스 | 프레스, Press |
| 소결 | 소결, Sintering |
| CVD | CVD |
| PVD | PVD, UL, LUL |
| 포장기 | 포장기, Packing, 포장 |
| 검사기 | 검사기, Inspection, F1, F2 |
| 호닝형상 | 호닝, HIM, Honing |
| 연삭 | 연삭, Grinding |
| CBN | CBN |
| 마스크자동기 | 마스크, Mask |
| 교정 | 교정 |

### 1-2. 복수 장비유형 등록 (중복 허용)
- 하나의 품목이 여러 장비 프로젝트에 사용된 경우, **모든 해당 장비유형을 등록**한다.
- 예: 서보모터가 프레스/검사기에 사용되면 `["프레스", "검사기"]`

### 1-3. 개선/개조/교체 프로젝트 포함 규칙
- **개선, 개조, 교체, 수리, 보수** 프로젝트도 장비유형 분류에 포함한다.
- 개조 프로젝트에서 감지된 장비유형도 동일하게 등록한다.
- 예: "교세라 포장기 1호기 개조" → 포장기에 포함
- 예: "[선제작] HIM-F1 #21-2 개조" → 호닝형상에 포함

### 1-4. 핸들러 장비유형 폐지 (2026-04-09)
- **"핸들러"는 장비유형으로 사용하지 않는다.**
- 키워드 "핸들러", "Handler"는 장비유형 매핑에서 **무시**.
- 핸들러만 단독 등록된 품목(66건) → **사용예정 제외 처리** (장비유형 빈배열, 리스트 유지).
- 기존 `["프레스", "핸들러"]` → `["프레스"]`로 변경 (핸들러만 제거).
- 기존 `["핸들러"]` 단독 → 장비유형 빈배열 처리.

### 1-5. 검사기-호닝형상 중복 처리
- "검사기(6)/호닝형상(6) 중복" 상태 → 발주이력 기반으로 `["검사기", "호닝형상"]` 명확히 분리 등록. (38건)
- HIM 프로젝트 → 호닝형상
- F1/F2 프로젝트 → 검사기

### 1-6. 연삭핸들러 유지
- "연삭핸들러"는 독립 장비유형으로 **그대로 유지**한다. (연삭으로 분리하지 않음)
- 데이터 형식만 수정: 문자열 `"연삭핸들러(1)"` → 리스트 `["연삭핸들러"]`

## 2. 제외/특수 처리 규칙

### 2-1. CS성 품목 (83건)
- CS/유지보수 전용 품목은 별도 표시 (CS성 플래그).
- 이전 CS제외 34건 + 신규 추가 49건 = 총 83건.
- 기준: 최근 발주이력이 CS성이고, **3년 이내 프로젝트 발주가 없는** 품목.
- CS성 키워드: CS, C/S, 충돌, 수리, 보수, 교체, 고장, 파손, 무상, AS, 긴급, 요청건
- **장비유형이 여러 개 등록되어 있어도** 실제 발주이력이 CS성뿐이면 제외 대상.
- 예: MADHT1505BA1 — 장비유형 8개 등록이지만 최근이력 CS, 3년 내 프로젝트 발주 없음 → 제외.

### 2-2. Cell Press 제외 (44건)
- "한국야금 프레스 Cell #1" 등 Cell 프로젝트 전용 품목 44건.
- 이 품목들은 일반 프레스 장비와 별도 분류.

### 2-3. Plan Only 품목 (78건)
- 구매계획만 있고 실제 발주가 없는 품목 78건.
- "공용자재 구매" 등 비프로젝트성 계획만 있는 경우 해당.

### 2-4. 비프로젝트성 발주 처리
- "20XX 공용자재 구매", "비프로젝트", "무상 자재" 등 → 장비유형 분류에서 제외.
- 단, 해당 품목의 다른 프로젝트 발주 기반 장비유형은 유지.

### 2-5. 프로젝트 예외 미배정 처리 (2026-04-09)
- 품목에 최근 프로젝트가 등록되어 있으나, 2번째/3번째 프로젝트에 예외가 있는 경우:
  - 최근 프로젝트는 그대로 유지
  - **장비유형은 원본 그대로 유지** (검사기 등 실제 사용 장비유형 보존)
  - 장비유형 자동 할당 대상에서만 제외 (혼동 방지)
- 예: SS-540A-V1 — 장비유형 `["검사기"]` 유지, 55348 — 장비유형 `["호닝형상"]` 유지
- 이 품목들은 장비유형 자동 할당 대상에서 제외.

### 2-6. 제외 키워드 프로젝트 자동 변경 (2026-04-09)
- JS `EXCLUDE_PJT` 키워드(`개발, 판매, 무상, 교체, 개조, 부품, 수리, 소모품, 소모성`)가 포함된 프로젝트 → 사용예정에서 제외됨.
- **장비유형이 있는 품목**의 프로젝트가 제외 키워드에 걸리면, 발주이력에서 **3년 이내 비제외 프로젝트**를 찾아 자동 변경.
- 공용자재/비프로젝트/미배정 프로젝트도 대체 대상에서 제외.
- 대체 프로젝트가 없으면 원본 유지 (JS가 사용예정 제외 처리).
- 적용 건수: 201건 (2026-04-09 기준)

### 2-7. 다수 장비 범용 품목 필터링 제외 (2026-04-09)
- JS `recalcAll()`은 프로젝트명에서 장비유형 키워드를 추출하여, BOM 장비유형과 교차 필터링함.
- 예: 장비유형 `["CVD","PVD","검사기","포장기","프레스"]`인데 프로젝트가 "한국야금 검사기 F1" → 검사기만 남김.
- **장비유형 2개 이상** 등록된 범용 품목(1,103건)은 이 필터링을 건너뛰어 전체 장비유형 반영.
- `gen_full_list.py`가 HTML 생성 시 JS에 `MULTI_EQUIP` Set을 패치하여 자동 적용.

### 2-8. 장비유형 수동 오버라이드 (2026-04-09)
- 자동 분류 오류가 확인된 품목은 수동으로 장비유형을 지정한다.
- 55348: 호닝형상 전용 품목 — `["검사기", "호닝형상"]` → `["호닝형상"]` (검사기 분류 오류 수정)
- MSMF042L1T2: 프로젝트 "연삭기개발-핸들러 (구매조건부)" → "아크시스 프레스 #2 (Kob,NSX2-25A)" (제외 키워드 회피)

### 2-10. 대구텍 키엔스검사자동화 전용 품목 제외 (2026-04-09)
- 대구텍 키엔스검사자동화 프로젝트 전용 품목은 Cell Press와 동일하게 사용예정에서 제외한다.
- 전용 여부 판단: 발주이력 기준 3년 이내 다른 장비 프로젝트에 사용된 적 없는 품목.
- 전용 34건: 장비유형 빈배열 처리 (`DAEGU_KEYENCE` Set, `gen_full_list.py`)
- 범용 5건: 다른 프로젝트에도 사용되므로 제외하지 않고, `PROJ_OVERRIDE`로 범용 프로젝트명 변경.
  - MSMF042L1T2 → 아크시스 프레스 #2 (Kob,NSX2-25A)
  - 4080ENDCAP → 몰디노 리팔레팅 #1
  - 50-02-55-0 → 사천신공 2jaw Gripper(MGT)
  - HXN6-2 → 한국야금 검사기 F1 (선제작#1)
  - LVBM20-50 → 한국야금 소결 #4
  - ZFC54-B → MMC 기후 프레스 #1

### 2-12. 부품교체 CS건 감지 (2026-04-09)
- 프로젝트명에 **장비 키워드 + 부품 키워드**가 동시 포함된 경우 → 부품교체 CS건으로 판단.
- 장비 키워드: 검사기, PVD, CVD, 프레스, 포장기, 소결, 호닝, 연삭, 그라인더
- 부품 키워드: 볼스크류, 모터, 실린더, 센서, 벨트, 베어링, 밸브, 스핀들, 카메라, 렌즈, 케이블, PCB, 지그, LM 등
- 예: "한국야금 검사기 #4 언로딩 Z축 볼스크류" → 검사기(장비) + 볼스크류(부품) → CS건
- **범용 품목**: 3년 이내 비CS 프로젝트가 있으면 해당 프로젝트로 자동 변경 (5건)
- **전용 품목**: 3년 이내 비CS 프로젝트가 없으면 사용예정 제외 (32건)

### 2-13. 공용자재(미배정) 프로젝트 자동 변경 (2026-04-09)
- "공용자재" 키워드가 포함된 프로젝트 → 제외 키워드 프로젝트와 동일하게 3년 이내 대체 프로젝트 탐색.
- 대체 프로젝트가 있으면 자동 변경, 없으면 원본 유지.

### 2-14. 발주이력 프로젝트 코드 정리 (2026-04-10)
- MCA210T/MAD111T JSON 데이터에서 아래 프로젝트 코드 접두사 데이터를 삭제한다:
  - **CSZ**: C/S 자재 (CS 수리/교체용)
  - **DPRN**: 개발용 자재
  - **SHZ**: 공용자재/소모성 자재
  - **PTZZ**: 고객사 단품/개조
- 이유: 이 프로젝트들은 장비유형 판단에 노이즈가 되므로, 발주이력에서 제거하여 데이터 품질 개선.
- 적용: `MCA210T_발주현황_10년.json` (270,227 → 236,913건), `MAD111T_재고감안_10년.json` (92,663 → 89,080건)

### 2-15. 프로젝트 참조 소스 (2026-04-10)
- 프로젝트 자동 변경 시 참조 데이터: **MCA210T + MAD111T 원본 JSON만 사용**.
- `multi_project_items.json`은 **더 이상 참조하지 않음** (폐기).
- `find_alt_project()`: MCA210T 발주이력(38,296개 품목) + MAD111T 재고감안(1,848개 품목) 통합 검색.

### 2-11. "제외"의 정의
- **"제외"는 리스트에서 품목을 삭제하는 것이 아니다.**
- 장비유형(`r[9]`)을 빈배열(`[]`)로 설정하여 **사용예정에서만 제외**.
- JS `recalcAll()` 함수가 장비유형 배열 기반으로 사용예정 수량/금액을 동적 계산.
- **전체 품목수(4,106건)는 항상 유지**되어야 한다.

## 3. 데이터 재생성 절차

### 3-1. ERP 원본 데이터 추출 시
1. ERP에서 현재고현황 + 구매진행현황 raw 데이터 추출
2. 발주이력(PO)에서 프로젝트명 기반 장비유형 매핑 (규칙 1-1 ~ 1-6 적용)
3. 제외 규칙 적용 (규칙 2-1 ~ 2-4)
4. 결과를 `erp_data.json`에 저장

### 3-2. 수동 오버라이드 적용
- `erp_data.json` 생성 후 아래 오버라이드 파일 순서대로 적용:
  1. `cs_only_items.json` — CS성 품목 플래그
  2. `cell_press_removed.json` — Cell Press 제외
  3. `plan_only_items.json` — Plan Only 표시
  4. `cs_type_items.json` — CS성 확장 목록

### 3-3. 검증 체크리스트
- [ ] 전체 품목수 확인 (현재 기준: 4,106개)
- [ ] 장비유형 빈값 품목 0건 확인 (비프로젝트 제외)
- [ ] 복수 장비 사용 품목의 장비유형 완전성 확인
- [ ] 개선/개조 프로젝트 장비유형 반영 확인
- [ ] CS성/Cell Press/Plan Only 제외 항목 유지 확인
