크로스주제 연결의 의미

항목포장혼입검사 이슈분말검사 이슈
현상경면 인서트 이미지 밝기 비대칭Burr 측면 시인 불가
원인광축 중심과 조명 중심 편차기존 조명 구성(동축·T-BLU) 한계
공통 패턴광학계 조명 불균형·불충분광학계 조명 불균형·불충분
담당 팀비전검사팀비전검사팀
발생 시기2026-01, 2025-022025년

기존 벡터 RAG와의 차이:

  • 벡터 RAG: 각 문서를 임베딩 → "밝기 비대칭"과 "Burr 불가"는 다른 키워드라 유사도 낮음
  • GraphRAG: 두 이슈가 모두 Component{type:'조명'} → INVOLVES_COMPONENT → Issue 패턴으로 연결 → 명시적 탐색으로 발견

실제 활용 시나리오: 엔지니어가 "포장 검사 장비 조명 문제 관련 사례 있나요?"라고 질의하면, 그래프 탐색으로 포장 담당자 모르게 분말 검사에서 같은 팀이 이미 해결한 유사 케이스를 즉시 연결.

5. 온톨로지 v1.1 확정 스키마

5.1 노드 유형

(:Customer   {name, alias, country, region})
(:Equipment  {name, model, type, location, customer, topic})

5.2 관계 유형

(Customer)  -[:OWNS]----------------> (Equipment)
(Equipment) -[:HAS_ISSUE]-----------> (Issue)
(Issue)     -[:SIMILAR_TO]----------> (Issue)          ← cross_topic 플래그 지원
(Issue)     -[:RESOLVED_BY]---------> (Resolution)
(Issue)     -[:INVOLVES_COMPONENT]--> (Component)
(Equipment) -[:USES_COMPONENT]------> (Component)
(Product)   -[:INVOLVED_IN]---------> (Issue)
(Process)   -[:HAS_SUBPROCESS]------> (Process)
(Process)   -[:USES_TOOL]-----------> (Tool)
(Equipment) -[:USES_PROCESS]--------> (Process)
